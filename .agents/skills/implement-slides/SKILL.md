# Skill: `implement-slides`

This skill takes a slide specification plan generated by `planning-slides` and builds the presentation HTML file using the **Reveal.js + dist/main.css + dist/main.js** component system of Universidad Icesi, automatically exporting the final **PDF slide deck** (`slides/<topic>/<topic>.pdf`).

---

## 📥 Parameters & Usage

- **`spec_file_path`**: Absolute or relative path to the plan specification markdown file saved under `.agents/spec/slides-plan/`.

Example usage:
```bash
/implement-slides .agents/spec/slides-plan/nestjs-plan.md
```

---

## ⚙️ Execution Guidelines

### 1. Read & Validate the Specification
1. Open and parse the provided spec file (`spec_file_path`).
2. Identify the target topic name, total slide count, HTML output path (`slides/<topic>/<topic>.html`), PDF output path (`slides/<topic>/<topic>.pdf`), and required assets.
3. Read `.agents/formatting_rules.md` before writing any slide HTML to ensure all typography, contrast, and margin rules are followed.

### 2. File & Directory Setup
1. Ensure the destination directory exists: `slides/<topic>/`.
2. Ensure the assets directory exists: `slides/<topic>/assets/`.
3. Create any custom SVGs or icons requested in the plan before assembling the HTML.

### 3. HTML Slide Generation Rules

#### A. Component Functions (Mandatory)
Use ONLY functions from `dist/main.js`:
- `icesi.titleSlideA(title, subtitle)` — `subtitle` can include `<span class="slide-footer-tag">` for institution labels with proper white-on-blue contrast.
- `icesi.slideStandard(title, content)`
- `icesi.slideTwoCols(title, col1, col2)`
- `icesi.slideThreeCols(title, col1, col2, col3)`
- `icesi.slideFourCards(title, c1title, c1, c2title, c2, c3title, c3, c4title, c4)`
- `icesi.slideSidebarLeft{Orange|Blue|Purple}(title, content, sidebarVisual)` — see sidebar rules below
- `icesi.slideStripeTop{Left|Right}(title, content)`
- `icesi.sectionSlide{A|B|C|E|EBlue|EGreen|EYellow|EOrange}(title)`
- `icesi.mermaid(code)` — for inline diagrams

#### B. No Ad-hoc Inline Styles
Do not add custom inline CSS `style=""` overrides to slide elements. Any new visual pattern must use existing CSS classes from `dist/main.css`.

#### C. Typography Hierarchy (Critical)
Always maintain the hierarchy: `Slide Title (h2, 38px)` > `Sub-heading (h3, 22px)` > `Body (p/li, 19px)` > `Card label (14px uppercase)`.
- **Never** write `<h3>` content in the same font-size range as the slide `h2` title.
- **Card headers** (`.card-header`) are the **smallest** elements — category labels, not titles.

#### D. Content Density Rules
- Do not leave more than 40% empty space.
- **Sidebar panel is a visual opportunity, not a dead zone**: When using `slideSidebarLeft*`, always provide meaningful content in the sidebar via `sidebarVisual`:
  - Mermaid diagram: `{ type: 'mermaid', code: 'flowchart ...' }` — contrast applied automatically.
  - SVG graphic: `{ type: 'graphic', html: '<svg>...</svg>' }` — fits in 429×540px.
  - Icon list: `{ type: 'icons', items: [{icon: '🔧', label: 'Feature Name'}] }` — up to 5 items.
  - Leave empty ONLY for explicitly minimalist transition slides.
- When content is concise (2–4 bullet points), use `icesi.mermaid(...)` or icon grids (`.icon-grid-2`, `.icon-grid-3`) to fill space.
- When a topic has many subtopics, prefer splitting into multiple slides over cramming content.

#### E. Cover Slide (titleSlideA) Rules
- Title: Use `<br>` for natural line breaks. Font auto-scales from 34px to 54px via `clamp()`.
- Subtitle (yellow box): Keep to 1–2 lines max.
- Institution/faculty label: **MUST** use `<span class="slide-footer-tag">Engineering Faculty — Universidad Icesi</span>` inside the subtitle `<p>`. Do NOT use `<small style="color:black">`.
- Example:
  ```javascript
  icesi.titleSlideA(
    'NestJS Framework:<br>Modern Server-Side Architecture',
    `Scalable, Enterprise-Grade Node.js with TypeScript
    <span class="slide-footer-tag">Engineering Faculty — Universidad Icesi</span>`
  )
  ```

#### F. Section Divider (sectionSlideE) Rules
- Section title uses `clamp(36px, 4.4vw, 57px)` — supports 2-line titles with subtitle.
- For 2-line titles with a subtitle, use:
  ```javascript
  icesi.sectionSlideEBlue(
    'Controllers & HTTP Endpoints<br><small style="font-size:0.5em; font-weight:normal;">Routing, payload extraction, and response handling</small>'
  )
  ```

#### G. Mermaid Diagram Contrast Rules
- **On dark backgrounds** (sectionSlideEBlue, sectionSlideEGreen, sectionSlideEOrange, titleSlideA, sectionSlideC): `base.css` automatically applies a white rounded background to the SVG. Still, **also explicitly set light node colors** using `%%{init}%%` or `classDef`:
  ```javascript
  icesi.mermaid(`
    %%{init: {'theme': 'base', 'themeVariables': {
      'primaryColor': '#5454E9',
      'primaryTextColor': '#ffffff',
      'lineColor': '#cccccc',
      'background': '#ffffff'
    }}}%%
    flowchart LR
      A[Client] --> B[Controller]
      classDef blue fill:#5454E9,stroke:none,color:#fff
      classDef green fill:#4CB979,stroke:none,color:#fff
      class A blue
      class B green
  `)
  ```
- **On white backgrounds** (slideStandard, slideTwoCols, slideSidebarLeft content area): Use brand colors freely.
- **Never** rely on default Mermaid gray nodes — always define node colors explicitly.

#### H. Table Safety Rules
- Tables must use component CSS classes — never write inline `width:` styles.
- Tables render correctly via `.slide-content table` / `.col table` styles in `base.css`.
- Never set a fixed px width on a table. Use `width: 100%` only.
- Never use `style="width: 1200px"` or similar — the slide canvas is only 1280px wide.

#### I. No Hyphenation
Use `<br>` explicitly in long titles to prevent unnatural word wrapping.

## 📄 PDF Generation & Quality Assurance Workflow (Mandatory Output)

The HTML file serves as the underlying structural medium, but **the ultimate primary deliverable of this skill is the PDF file (`slides/<topic>/<topic>.pdf`)**.

### Step 1: PDF Export Generation
Export the Reveal.js HTML slide deck directly to a single PDF document using `decktape`:
```powershell
npx -y decktape reveal http://localhost:<port>/slides/<topic>/<topic>.html slides/<topic>/<topic>.pdf
```
*(Or via direct print-pdf headless rendering).*

### Step 2: Quality Inspection Checklist
Inspect every slide in `slides/<topic>/<topic>.pdf` and verify ALL of the following:

| Check | Pass Criteria |
|---|---|
| **Text overflow** | Zero text cut off by slide boundaries or colored boxes |
| **Mermaid visibility** | All diagrams are clearly visible — no white-on-white or colored-on-same-color |
| **Typography hierarchy** | Slide title > sub-heading (h3) > body (li, p) > card label |
| **Sidebar usage** | Sidebar panel has visual content (not empty unless intentional) |
| **Table margins** | Tables do not touch the right or bottom slide edges |
| **Card structure** | Card header label (14px) is smaller than card body text (17px) |
| **Cover slide** | Institution label is a white-on-blue badge, not black text on blue |
| **Content density** | No slide is < 60% full; dense slides are split across multiple |

### Step 3: Refine & Re-verify
If any check fails, edit the HTML and re-export until all checks pass.
